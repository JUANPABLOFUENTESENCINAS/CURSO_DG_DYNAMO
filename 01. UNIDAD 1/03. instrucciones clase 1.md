# Clase 1: Fundamentos de Programación Visual en Dynamo 

Este documento contiene la guía práctica paso a paso de los conceptos fundamentales abordados en la primera sesión del curso. Cada sección describe la lógica operativa de los nodos básicos, tipos de datos y control de variables.

---

## 1. INTRODUCCIÓN A LOS NUDOS E INTERFACES

![Texto alternativo](NUDOS.PGN)

### 1. OBJETIVO DEL EJEMPLO
El objetivo de este ejercicio inicial es comprender la anatomía básica de un nodo en Dynamo, identificando los puertos de entrada (Inputs), los puertos de salida (Outputs) y cómo se transmite la información a través de los cables para generar una entidad geométrica básica (un punto) y visualizar sus datos.

### 2. EXPLICACION
El script presenta la estructura elemental de la programación visual:
* **Definición de Variables:** Se utilizan tres nodos de entrada. Un nodo `Number` estático asignado al eje **X**, y dos controles deslizantes `Number Slider` asignados de forma dinámica a los ejes **Y** y **Z**.
* **Procesamiento Geométrico:** Los tres cables convergen en el nodo `Point.ByCoordinates`. Este nodo procesa las tres variables numéricas simultáneamente para instanciar un punto en el espacio virtual tridimensional.
* **Visualización de Datos:** La salida del punto se conecta a un nodo `Watch` (Visor). Este nodo no altera la geometría, sino que extrae la información textual interna del elemento, mostrando las coordenadas precisas calculadas en formato de texto legible.

### 3. RECOMENDACIONES
* **Doble Clic para Visores:** El nodo `Watch` es una excelente herramienta de depuración en niveles básicos. usarlo cada vez que un nodo geométrico o numérico no muestre el resultado esperado para rastrear dónde se rompió el flujo.

## 2. TIPOS DE DATOS FUNDAMENTALES

![Texto alternativo](TDATOS.PGN)

### 1. OBJETIVO DEL EJEMPLO
El objetivo de este ejercicio es mapear y comprobar de forma aislada el comportamiento de los diferentes tipos de datos alfanuméricos y lógicos disponibles en Dynamo para Revit 2026, entendiendo cómo se estructuran antes de interactuar con elementos del modelo BIM.

### 2. EXPLICACION
En este bloque se realiza un muestreo de las variables estándar de programación conectadas directamente a nodos `Watch` para comprobar su salida limpia:
* **Datos Numéricos:** Se contrasta el nodo `Number` tradicional con los deslizadores discretos (`Integer Slider` para números enteros) y continuos (`Number Slider` para decimales).
* **Datos de Texto:** Se observa el uso del nodo `String` para almacenar palabras o frases sueltas, y cómo un `Code Block` puede almacenar texto escribiendo la cadena entre comillas dobles (`"hola mundo"`).
* **Datos Lógicos y Temporales:** Se presentan las variables condicionales `Boolean` (conmutador *True/False*) y el tipo de dato `DateTime.Date` que extrae formatos de tiempo y calendario.

### 3. RECOMENDACIONES
* **Eficiencia con Code Blocks:** Un `Code Block` puede reemplazar tanto a nodos de texto como a nodos numéricos simplemente haciendo doble clic en la pantalla y escribiendo un número o texto entre comillas. Esto ahorra espacio y memoria en scripts complejos.

## 3. MANEJO DE OPERADORES ARITMÉTICOS

![Texto alternativo](OPERADORES.PGN)

### 1. OBJETIVO DEL EJEMPLO
El objetivo de este ejercicio es aprender a ejecutar operaciones matemáticas básicas (Suma, Resta, Multiplicación y División) dentro de Dynamo utilizando la sintaxis condensada de los bloques de código (`Code Block`), reemplazando el uso de nodos matemáticos individuales.

### 2. EXPLICACION
El ejercicio demuestra cómo un solo `Code Block` puede actuar como un procesador de fórmulas algebraicas:
* **Declaración de Fórmulas:** En lugar de buscar nodos individuales para sumar o multiplicar, se configuran cuatro bloques de código independientes con las instrucciones básicas: `x + y;`, `x * y;`, `x - y;` y `x / y;`.
* **Generación Automática de Puertos:** Al declarar variables genéricas (`x` e `y`), Dynamo transforma automáticamente estos caracteres en puertos de entrada físicos en el costado izquierdo del nodo.
* **Alimentación y Cálculo:** Se conectan parejas de `Number Sliders` a las entradas generadas. El bloque procesa los operadores aritméticos de forma nativa y envía el resultado exacto hacia el nodo `Watch` final.

### 3. RECOMENDACIONES
* **Sintaxis Estricta:** Las omisión del punto y coma (`;`) al final de la fórmula dentro del `Code Block` es el error más común en los estudiantes de nivel básico. Sin este carácter, el script detendrá su ejecución.

## 4. MANEJO Y CONCATENACIÓN DE STRINGS (TEXTOS)

![Texto alternativo](STRIN.PGN)

### 1. OBJETIVO DEL EJEMPLO
El objetivo de este ejercicio es dominar la manipulación de cadenas de texto en Dynamo. Se busca aprender a fusionar información alfanumérica y modificar sus propiedades (longitud, mayúsculas/minúsculas), una habilidad crítica para la posterior automatización de nombres de familias y parámetros en Revit.

### 2. EXPLICACION
Esta sección aborda la gestión de textos desde dos metodologías equivalentes (Nodos vs. Código):
* **Concatenación por Nodos:** Se utiliza el nodo `String.Concat` para unificar tres entradas distintas: un texto base (`"Muros de "`), un número proveniente de un bloque de código (`10`) y un sufijo dimensional (`"cm"`), unificándolos en una sola oración.
* **Concatenación por Código:** Se replica la misma lógica anterior de manera directa en un `Code Block` utilizando el operador de suma (`+`) como herramienta de unión textual: `"Muro" + " " + "Portante";`.
* **Análisis y Transformación:** Se introducen nodos de propiedades de texto, como `String.Length` para contar el número exacto de caracteres de una cadena, y nodos de edición como `String.ToUpper` o `String.ToLower` para forzar textos a mayúsculas o minúsculas.

### 3. RECOMENDACIONES
* **Espacios en Blanco:** Al concatenar textos con variables numéricas, recuerda siempre añadir un espacio en blanco al final de la cadena de texto inicial (ej. `"Muros de "` en lugar de `"Muros de"`), de lo contrario, las palabras y los números quedarán pegados en el resultado final (`Muros de10cm`).

## 5. PROCESO ESTÁTICO VS. PROCESO PARAMETRIZADO

![Texto alternativo](PROCESS.PGN)

### 1. OBJETIVO DEL EJEMPLO
El objetivo final de esta sesión es contrastar directamente la diferencia operativa e intelectual entre el modelado tradicional rígido (Proceso Estático) y el diseño algorítmico adaptativo (Proceso Parametrizado), evidenciando el verdadero potencial de Dynamo para el diseño de infraestructura.

### 2. EXPLICACION
Se desarrollan dos enfoques metodológicos para resolver un mismo problema geométrico (crear un círculo en el espacio):
* **5.1 Proceso Estático:** Se emplean nodos rígidos de tipo `Number` con valores fijos grabados en su interior ($X=0, Y=0, Z=0$) y un radio invariable de $1$. Si el usuario requiere modificar la posición o la escala de la figura, se ve obligado a reescribir manualmente los valores numéricos dentro de cada nodo. Es una automatización pasiva.
* **5.2 Proceso Parametrizado:** Se reemplazan todos los números fijos por nodos dinámicos `Number Slider`. Al interconectar estos deslizadores al nodo `Point.ByCoordinates` y posteriormente al radio del `Circle.ByCenterPointRadius`, la geometría cobra vida. El usuario puede arrastrar los controles y ver cómo el círculo se desplaza y cambia de tamaño fluidamente en el espacio tridimensional.

### 3. RECOMENDACIONES
* **El Núcleo del Curso:** El objetivo de aprender Dynamo no es crear procesos estáticos (enfoque 5.1), sino estructurar siempre flujos parametrizados (enfoque 5.2). Diseñar pensando en variables permite que un solo script pueda adaptarse a múltiples proyectos y escenarios de ingeniería civil sin necesidad de volver a programar desde cero.
